## 一、货币系统

### 1.1 基础汇率

| 货币 | 价值 |
|------|------|
| 1 月石 | $0.01 USD |
| 100 月石 | $0.99 USD |
| 1000 月石 | $9.99 USD |

**设计原则**：
- 月石是唯一的硬通货
- 所有消费都以月石计价
- 避免多货币系统的复杂性

### 1.2 月石获取途径

#### 免费获取

| 途径 | 月石/次 | 频率 | 月累计 |
|------|---------|------|--------|
| 每日签到 (7日平均) | 7 | 每日 | ~200 |
| 看广告 | 5 | 3次/日 | ~450 |
| 新手任务 | 100 | 一次性 | 100 |
| 邀请好友 | 50 | 无限 | - |
| 特殊活动 | 10-50 | 不定期 | - |

**免费用户月获取上限**：约 750 月石 (≈$7.50)

#### 付费获取

| 充值档位 | 价格 | 月石 | 首充加成 | 单价 |
|----------|------|------|----------|------|
| 小额 | $0.99 | 100 | +100 | $0.0099 |
| 基础 | $4.99 | 500 | +200 | $0.0100 |
| 标准 | $9.99 | 1000 | +300 | $0.0100 |
| 进阶 | $19.99 | 2000 | +500 | $0.0100 |
| 豪华 | $49.99 | 5000 | +1000 | $0.0100 |
| 至尊 | $99.99 | 10000 | +2000 | $0.0100 |

**首充双倍**：首次充值任意档位，月石翻倍（仅首次）

### 1.3 月石消费场景

| 消费类型 | 价格范围 | 说明 |

| 连续天数 | 额外奖励 |
|----------|----------|
| 7 天 | 🎁 随机 Tier 1 礼物 |
| 14 天 | 💎 50 月石 |
| 21 天 | 🎁 随机 Tier 2 礼物 |

### 4.4 补签机制

| 补签时机 | 消耗 | 限制 |
|----------|------|------|
| 24 小时内 | 5 月石 | 无限次 |

**设计逻辑**：
- 补签花费略高于当日奖励，保持心理成本
- 限制补签次数，鼓励每日登录
- 超时无法补签，增加紧迫感

## 附录：数据库 Schema

### 签到系统

```sql
-- 签到记录表
CREATE TABLE daily_check_ins (
    id UUID PRIMARY KEY,
    user_id VARCHAR(64) NOT NULL,
    check_in_date DATE NOT NULL,
    streak_day INT DEFAULT 1,          -- 连续签到天数 (1-7循环)
    reward_amount INT,                 -- 领取的月石数量
    bonus_gift_type VARCHAR(64),       -- 额外礼物类型
    is_makeup BOOLEAN DEFAULT FALSE,   -- 是否补签
    makeup_cost INT DEFAULT 0,         -- 补签花费
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, check_in_date)
);

-- 签到状态表
CREATE TABLE check_in_streaks (
    user_id VARCHAR(64) PRIMARY KEY,
    current_streak INT DEFAULT 0,       -- 当前连续天数
    current_cycle_day INT DEFAULT 0,    -- 当前周期第几天 (1-7)
    longest_streak INT DEFAULT 0,       -- 历史最长连续
    last_check_in DATE,
    total_check_ins INT DEFAULT 0,
    monthly_makeup_count INT DEFAULT 0, -- 本月补签次数
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 订阅系统

```sql
-- 用户订阅表
CREATE TABLE user_subscriptions (
    id UUID PRIMARY KEY,
    user_id VARCHAR(64) NOT NULL,
    tier VARCHAR(32) NOT NULL,          -- silver_pass, vip_gold
    status VARCHAR(32) NOT NULL,        -- active, paused, cancelled
    started_at TIMESTAMP NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    auto_renew BOOLEAN DEFAULT TRUE,
    platform VARCHAR(32),               -- ios, android, web
    transaction_id VARCHAR(128),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 订阅福利领取记录
CREATE TABLE subscription_claims (
    id UUID PRIMARY KEY,
    user_id VARCHAR(64) NOT NULL,
    claim_date DATE NOT NULL,
    moon_stones INT,
    bonus_gift_type VARCHAR(64),
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, claim_date)
```
